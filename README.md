# E-LOTs

デジタルと紙を組み合わせた、文化祭向けおみくじWebアプリ

[公開URL](https://shimataiyaki.github.io/e-lots/)

---

## 概要
E-LOTsは、来場者のスマートフォンで抽選を行い、その結果に応じて紙のおみくじを手渡す仕組みのWebアプリです。  
デジタル完結ではなく、**必ず対面のコミュニケーションが発生する設計**を特徴としています。

---

## 特徴
- ブラウザ上で完結するシンプルな抽選システム
- 紙のおみくじと連動した運用設計
- モバイル前提のUI設計（文化祭運用に最適化）
- 縦書き・札UIによる和風デザイン
- ネットワークや機材トラブルに強い静的構成（GitHub Pages）

---

## 使い方（来場者向け）
1. スマホでアクセス  
2. 好きな札をタップ  
3. 表示された番号をスタッフに見せる  
4. 対応する紙のおみくじを受け取る  

---

## システムの流れ
来場者は自身のスマートフォンで抽選を行い、その結果をもとにスタッフが紙のおみくじを手渡します。

```

[来場者のスマホ] → [GitHub Pages] → [ブラウザ上で抽選]
↓
[スタッフが番号を照合]
↓
[紙のおみくじを手渡し

```

---

## 技術構成
- **フロントエンド**：HTML5 / CSS3 / JavaScript（ES6）
- **ホスティング**：GitHub Pages
- **開発環境**：VS Code / Cursor
- **開発手法**：AI支援による高速プロトタイピング

---

## 技術解説

### 抽選ロジック
- 漢数字（`一`〜`十八`）からランダムに選択  
- 「四」「十四」は文化的配慮から除外  
- 等確率で16種類を維持  

### UI/UX設計
- CSSによる縦書き（`writing-mode: vertical-rl`）
- ホバー時の浮遊アニメーション
- モーダルによる結果表示（手動閉鎖）
- 連打防止（`pointer-events: none`）
- **Gen.5にてUIを再設計し、独自構成へ移行**

### 運用設計（コア設計）
- 結果は「番号のみ」表示
- 運勢はあえて表示しない
- スタッフが紙と照合して手渡す

▶デジタル単体で完結させず、**対話を設計に組み込んでいる**

## 課題と対応

| 課題 | 解決策 |
| :--- | :--- |
| 前の結果が残る | モーダル閉鎖時に完全リセット |
| 「四」が不吉 | 抽選対象から除外 |
| デザイン不統一 | UI再設計により統一 |

## 命名について
リポジトリ名 `e-lots` は、電子（e）と抽選（lots）を組み合わせた名称です。  
本プロダクトは「デジタルおみくじ」として公開しています。
